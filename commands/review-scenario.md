# 시나리오 리뷰

입력: $ARGUMENTS

---

## 1. 리뷰 대상 확인

### 입력 처리
- 파일 경로가 주어진 경우: 해당 파일을 읽어서 리뷰
- 제품명+버전이 주어진 경우: `data/scenarios/{제품명}_{버전}_*_scenario.md` 패턴으로 검색

---

## 2. 기획서/정책서 대비 검토

### 2.1 커버리지 검토
- 기획서/정책서의 모든 기능이 시나리오에 포함되었는지 확인
- 누락된 기능/케이스 식별

### 2.2 정확성 검토
- 시나리오의 조건-결과가 기획서/정책서와 일치하는지 확인
- 잘못 해석되거나 변형된 내용 식별

### 2.3 조건-결과 모순
- "A 불가" 조건인데 "A 후 동작" 정의된 경우
- 예: "상담 내역 있으면 고객 삭제 불가" + "삭제 후 상담 이력 유지"

### 2.4 상태 흐름 오류
- 도달 불가능한 상태 정의
- 순환 참조 (A→B→A 무한루프)
- 상태 전이 누락 (A에서 B로 갈 수 없는데 B 상태 존재)

### 2.5 섹션 간 불일치
- 같은 기능이 다른 시나리오에서 다르게 정의
- 권한/역할별 동작 충돌

### 2.6 경계 조건 누락
- 0건, 최대값, 빈 값 처리 미정의
- "이상/이하" vs "초과/미만" 혼용

### 2.7 용어 일관성
- 기획서 용어와 시나리오 용어가 다른 경우
- 같은 개념에 다른 용어 사용

---

## 3. 시나리오 품질 검토

### 3.1 테스트 가능성
- 각 시나리오가 검증 가능한 형태인지 확인
- 모호한 표현 식별 (예: "적절히 표시", "정상 동작")
- 기대 결과가 명확한지 확인

### 3.2 시나리오 간 중복/충돌
- 동일한 케이스가 여러 시나리오에 중복 정의
- 시나리오 간 결과가 상충하는 경우

### 3.3 우선순위 적정성
- P1/P2/P3 분류가 기준에 맞는지 확인
  - P1: 핵심 기능, 데이터 무결성, 보안
  - P2: 주요 기능, 일반 사용 흐름
  - P3: UI/UX 세부사항, 부가 기능

### 3.4 형식 준수
- 마크다운 구조 (H1 > H2 > H3 > H4) 준수 여부
- `{조건/동작} → {결과}` 형식 통일 여부
- 번호 리스트 사용 여부
- 파일명 규칙 준수 여부

---

## 4. 출력 형식

```markdown
## 시나리오 리뷰 결과

**대상:** {파일명 또는 제품명 v버전}
**리뷰일:** YYYY-MM-DD
**참고 문서:** {기획서}, {정책서}

---

### 커버리지 요약
| 구분 | 기획서 기능 수 | 시나리오 반영 수 | 커버리지 |
|------|--------------|----------------|---------|
| 전체 | {N} | {N} | {N}% |

### 발견된 문제
| # | 유형 | 위치 | 내용 | 심각도 |
|---|------|------|------|--------|
| 1 | 커버리지 누락 | {섹션} | {내용} | 높음/중간/낮음 |
| 2 | 정확성 오류 | {섹션} | {내용} | 높음/중간/낮음 |
| 3 | 모호한 표현 | {섹션} | {내용} | 높음/중간/낮음 |

### 심각도 기준
- **높음**: 기능 누락, 조건-결과 오류, 기획서와 불일치
- **중간**: 경계 조건 누락, 우선순위 부적절, 용어 불일치
- **낮음**: 형식 미준수, 표현 모호, 중복 정의

### 개선 권장사항
1. {권장사항}
2. {권장사항}

### 수정 필요 시나리오
- {시나리오 번호}: {수정 내용}
```
