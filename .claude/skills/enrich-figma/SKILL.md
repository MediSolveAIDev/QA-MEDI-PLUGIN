# Figma 기반 시나리오 보강

입력: $ARGUMENTS

---

## 0. 개요

Figma에서 export된 디자인 이미지(PNG)와 설명 텍스트(descriptions.txt)를 읽어,
기존 시나리오에 누락된 UI 요소, 문구, 상태별 화면을 보강하는 스킬이다.

### 입력 방식
1. **figma_output 폴더 경로**: `figma_output/say_admin_1.3.0`
2. **프로젝트명_버전**: `say_admin_1.3.0` → `figma_output/say_admin_1.3.0/` 자동 탐색
3. **프로젝트명만**: `SAY` → `figma_output/` 하위에서 최신 폴더 자동 탐색

### 필수 파일 구조
```
figma_output/{프로젝트}_{버전}/
├── summary.json          ← 섹션별 디자인/설명 매핑 (필수)
├── descriptions.txt      ← 전체 텍스트 설명 (필수)
└── *.png                 ← 디자인 이미지 파일들
```

## 1. 사전 검증

### 1.1 figma_output 폴더 확인
```
1. summary.json 존재 여부 확인
2. descriptions.txt 존재 여부 확인
3. PNG 파일 개수 확인
```

**폴더/파일 없을 경우:**
```
⚠️ figma_output 폴더가 없거나 필수 파일이 누락되었습니다.

Figma export를 먼저 실행해주세요:
  python tools/figma_extract.py

또는 figma_output/{프로젝트}_{버전}/ 폴더에 다음 파일이 필요합니다:
  - summary.json (섹션별 디자인/설명 매핑)
  - descriptions.txt (텍스트 설명)
  - *.png (디자인 이미지)
```

### 1.2 대상 시나리오 확인
- `data/scenarios/` 에서 해당 프로젝트/버전의 시나리오 파일 탐색
- 시나리오 파일이 없으면 사용자에게 확인:
  ```
  ⚠️ 보강할 시나리오 파일을 찾을 수 없습니다.
  시나리오 파일 경로를 직접 지정하거나, /write-scenario로 먼저 시나리오를 작성해주세요.
  ```
- 시나리오 파일이 여러 개면 목록을 보여주고 선택 요청

## 2. Figma Export 분석

### 2.1 summary.json 파싱
summary.json의 구조:
```json
[
  {
    "name": "섹션명",
    "id": "figma_node_id",
    "designs": [
      { "name": "디자인명", "id": "node_id", "file": "파일명.png" }
    ],
    "descriptions": [
      { "name": "설명명", "id": "node_id", "text": "설명 텍스트" }
    ]
  }
]
```

**파싱 규칙:**
1. 각 섹션(section)을 순회
2. `designs` 배열에서 PNG 파일 목록 추출
3. `descriptions` 배열에서 텍스트 설명 추출
4. 섹션명 → 시나리오의 대분류/중분류와 매핑

### 2.2 디자인 이미지 분석 (PNG 읽기)
- 각 섹션의 PNG 파일을 Read 도구로 읽어 시각적으로 분석
- **분석 관점:**
  - UI 요소 식별 (버튼, 입력 필드, 리스트, 모달, 토스트 등)
  - 상태별 화면 구분 (기본/호버/활성/비활성/에러/빈 상태)
  - 화면 간 전환 흐름 (어디서 어디로 이동하는지)
  - 문구/라벨 텍스트 확인

### 2.3 텍스트 설명 분석
- descriptions.txt 또는 summary.json의 descriptions.text 필드 분석
- **추출 항목:**
  - 위치 정보 (`# 위치 : ...`)
  - 번호 매김된 UI 요소 (`[1]`, `[2]`, ...)
  - 동작/인터랙션 정의 (클릭 시, 호버 시, 선택 시)
  - 상태 조건 (데이터 없음, 값 있음, 비활성화 등)
  - 에러/예외 처리 (실패 시, 오류 시)

## 3. 시나리오 보강 수행

### 3.1 매핑 (Figma 섹션 ↔ 시나리오 섹션)

1. summary.json의 각 섹션명을 기존 시나리오의 H1/H2/H3 헤딩과 매핑
2. 매핑 결과를 테이블로 출력:

```
## Figma ↔ 시나리오 매핑

| Figma 섹션 | 시나리오 섹션 | 매핑 상태 |
|-----------|-------------|----------|
| AI 가이드 대시보드 | 1.1 AI 가이드 대시보드 | ✅ 매핑됨 |
| AI 품질 점검 | 2.1 AI 품질 점검 | ✅ 매핑됨 |
| 예외 케이스 | (없음) | ⚠️ 시나리오에 없음 |
```

3. 매핑 안 되는 섹션은 **신규 시나리오 추가 대상**으로 분류

### 3.2 크로스 체크 (5가지 관점)

각 매핑된 섹션에 대해 다음 5가지를 검토:

#### ① UI 요소 누락
- Figma 디자인에 있는 버튼/필드/영역이 시나리오에 빠져 있는지
- 예: Figma에 [초기화] 버튼이 있는데 시나리오에 초기화 동작이 없음

#### ② 문구 일치
- Figma description의 문구와 시나리오의 문구가 다른지
- 예: Figma "아직 등록된 AI 가이드가 없어요" vs 시나리오 "등록된 가이드가 없습니다"
- **Figma 문구를 기준으로** 시나리오 수정

#### ③ 상태별 화면 커버리지
- Figma에 정의된 상태(기본/호버/활성/비활성/에러/빈 상태)가 시나리오에 모두 반영되었는지
- 예: Figma에 "데이터 없음 상태" 디자인이 있는데 시나리오에 빈 상태 시나리오가 없음

#### ④ 인터랙션/전환 누락
- Figma description에 정의된 동작(클릭 시, 호버 시)이 시나리오에 반영되었는지
- 예: Figma "[품질 점검] 클릭 시 AI 품질 점검 화면 이동" → 시나리오에 화면 이동 동작 있는지

#### ⑤ Figma에만 있는 정보
- 시나리오 대분류에 매핑되지 않는 Figma 섹션
- 시나리오에 없는 새로운 UI 요소나 동작

### 3.3 보강 실행

크로스 체크 결과를 바탕으로 시나리오 수정:

1. **누락된 시나리오 추가**: Figma에만 있는 내용을 새 시나리오 항목으로 추가
2. **문구 수정**: Figma 기준으로 문구 통일
3. **상태별 케이스 보완**: 빠진 상태(빈 상태, 에러 상태 등) 시나리오 추가
4. **인터랙션 보완**: 누락된 동작/전환 시나리오 추가

### 3.4 디자인 스펙 필터링 (TC 작성 규칙 준수)

시나리오 보강 시 다음은 **포함하지 않음**:
- 픽셀값 (w222, h100 등)
- Hex 색상 코드 (#5442FF 등)
- 디자인 토큰 (Gray 500, Main Blue 등)
- 스타일 스펙 (Filled, Outline 등)

대신 **기능적 상태로 표현**:
```
❌ 배경 색상 Gray-300(#E3E3E3)에서 Main Blue(#5442FF)로 변경됨
✅ 비활성화 상태에서 활성화 상태로 변경됨

❌ w350 영역을 초과할 경우 말줄임(...) 처리
✅ 가이드명이 영역을 초과할 경우 말줄임 처리됨
```

## 4. 출력

### 4.1 보강 결과 리포트

먼저 크로스 체크 결과를 리포트로 출력:

```markdown
## Figma 보강 결과 리포트

### 요약
- 분석한 Figma 섹션: {N}개
- 매핑된 시나리오 섹션: {N}개
- 신규 추가: {N}건
- 문구 수정: {N}건
- 상태 보완: {N}건
- 인터랙션 보완: {N}건

### 변경 상세

#### 신규 추가
| # | Figma 섹션 | 추가된 시나리오 | 근거 |
|---|-----------|--------------|------|
| 1 | 예외 케이스 | 3.1 로딩/에러 처리 | Figma에 예외 케이스 디자인 존재 |

#### 문구 수정
| # | 위치 | 변경 전 (시나리오) | 변경 후 (Figma 기준) |
|---|------|-----------------|-------------------|
| 1 | 1.2.3 빈 상태 | 등록된 가이드가 없습니다 | 아직 등록된 AI 가이드가 없어요, 새로 등록해보세요 |

#### 상태 보완
| # | 시나리오 섹션 | 추가된 상태 | Figma 디자인 |
|---|------------|-----------|------------|
| 1 | 1.1 대시보드 | 데이터 없음 상태 | AI_가이드_대시보드_값 없음.png |

#### 인터랙션 보완
| # | 시나리오 섹션 | 추가된 동작 | Figma 근거 |
|---|------------|-----------|-----------|
| 1 | 1.2 점검 항목 | [확인] 클릭 시 항목 제거 | description node 12883:19979 |

### 불일치 목록 (팀장 확인 필요)
| # | Figma | 시나리오/기획서 | 판단 |
|---|-------|-------------|------|
| 1 | "점검이 필요한 항목이 없어요" | 기획서에 해당 문구 없음 | Figma 기준 반영 |
```

### 4.2 보강된 시나리오 파일 저장

- 원본 시나리오 파일 백업: `{원본파일명}.backup.md`
- 보강된 시나리오 저장: 원본 파일 덮어쓰기
- 저장 경로: `data/scenarios/{프로젝트}_{버전}_{기능명}_scenario.md`

### 4.3 팀장 확인 요청

보강 완료 후 팀장에게 확인 요청:

```
## Figma 보강 완료

보강된 시나리오: data/scenarios/{파일명}
변경 요약: 신규 {N}건 / 수정 {N}건 / 상태 보완 {N}건 / 인터랙션 {N}건

불일치 {N}건은 Figma 기준으로 반영했습니다.
확인 후 시나리오 확정 진행하시겠습니까?
```

## 5. 결과물 문체 규칙

보강 시 추가/수정하는 시나리오 문장은 write-scenario 규칙을 따른다:

- **형식**: `{조건/동작} → {결과}`
- **긍정**: ~됨, ~적용됨, ~표시됨, ~유지됨, ~노출됨
- **부정**: ~되지 않음, ~불가함, ~없음
- **지양**: ~된다, ~합니다, ~해야 함 (서술형 혼용 금지)

---

## 6. 제약 사항

- Figma MCP API 호출 금지 (export된 로컬 파일만 사용)
- summary.json에 없는 노드 ID로 Figma API 요청 금지
- 디자인 스펙(색상, 크기, 폰트) 시나리오 포함 금지
- 기획서/정책서에 명시적으로 반하는 내용은 Figma 기준이라도 불일치로 보고 (팀장 판단)
